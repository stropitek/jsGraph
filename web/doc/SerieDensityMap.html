{SerieDensityMap} colorMapHSL( Array.<Object> colorStops , Number numColors [ , String method ] )

Generates a color map based on a serie of HSL(A) values.

Colors can scale linearly, logarithmically (enhances short range differences) or exponentially (enhances long range differences). One word of advice though. SVG being not canvas, jsGraph has to create a path for each color value of the color map. In other words, if you're asking for 16-bit coloring (65536 values), 65536 SVG paths will be created and your browser will start to suffer from it. As of now, all the colors in colorStops will be places at equal distances from each other between colorMapMin and colorMapMax set by autoColorMapBinBoundaries or setColorMapBinBoundaries

Parameters:
Name Type Attributes Default Description
colorStops Array.<Object> An array of objects, each having the following format: { h: [ 0-360], s: 0-1, l: 0-1, a: 0-1}
numColors Number The number of colors to compute.
method String <optional>
 "linear" The method to use to calculate the density map: linear, exp, or log
Returns:
{ SerieDensityMap } The current instance

{SerieDensityMap} setPxPerBin( Number pxPerBinX , Number pxPerBinY , Boolean weightedDensityMap )

Only calculates the density map upon redraw based on the current state of the graph. In this mode, a fixed number of pixels per bin is used to calculate the number of bins and fed into the calculation of the density map. In this method, the color map spans on the full scale of the density map values (i.e. a subrange cannot be defined, like you would do using SerieDensityMap#setColorMapBinBoundaries).
Parameters:
Name Type Description
pxPerBinX Number The number of x bins per pixels. Should be an integer, but technically it doesn't have to
pxPerBinY Number The number of y bins per pixels. Should be an integer, but technically it doesn't have to
weightedDensityMap Boolean Whether jsGraph should use weighted density mapping or not
See:
  • SerieDensityMap#calculateDensity
Returns:
{ SerieDensityMap } The current instance
